# Multi-Agent Orchestration Structure

## TEEI Website (Astro)

**Tech Stack**: Astro 5, React (islands), TailwindCSS, Markdown content, Multi-locale (en/uk/no)

**Purpose**: High-performance, SEO-optimized static website for TEEI brand presence, impact storytelling, and user onboarding. Built with Astro's partial hydration model and content collections for maximum performance and accessibility.

---

## Build & Test Commands

```bash
# Install dependencies
pnpm install

# Development server with hot reload
pnpm dev

# Type checking (strict mode)
pnpm typecheck

# Linting (ESLint + Prettier)
pnpm lint
pnpm format

# Build for production
pnpm build

# Preview production build locally
pnpm preview

# Run all tests
pnpm test

# Run tests in watch mode
pnpm test:watch

# SEO validation (meta tags, structured data, sitemaps)
pnpm seo:validate

# Accessibility audit (axe-core)
pnpm a11y:audit

# Performance metrics (Lighthouse)
pnpm perf:audit
```

- ❌ NEVER modify `astro.config.mjs` without running `pnpm build` to verify
- ❌ NEVER commit broken build (always test with `pnpm build` before pushing)
- ❌ NEVER add external scripts without reviewing performance impact
- ❌ NEVER create new React island without justification (prefer static Astro components)
- ❌ NEVER skip TypeScript validation (use strict mode)
- ❌ NEVER modify SEO metadata without testing with `pnpm seo:validate`
- ❌ NEVER commit changes without running accessibility audit (`pnpm a11y:audit`)
- ❌ NEVER store secrets or API keys in environment files visible in repo

#### ALWAYS (Required)
- ✅ ALWAYS run `pnpm build && pnpm preview` before submitting PR
- ✅ ALWAYS validate TypeScript (no `any` types, strict mode enabled)
- ✅ ALWAYS test multi-locale rendering for content changes
- ✅ ALWAYS include alt text for images and ARIA labels for interactive elements
- ✅ ALWAYS document new content collections in `src/content/config.ts`
- ✅ ALWAYS verify generated sitemap includes all locales
- ✅ ALWAYS test Lighthouse score (target: 90+)

**Input:** "Create dynamic impact story detail page with locale support"

**Output:**
```astro
---
// src/pages/[locale]/impact/[slug].astro
import { getCollection } from 'astro:content';
import BaseLayout from '../../../layouts/BaseLayout.astro';

export async function getStaticPaths() {
  const locales = ['en', 'uk', 'no'];
  const stories = await getCollection('impact-stories');

  return locales.flatMap(locale =>
    stories.map(story => ({
      params: { locale, slug: story.slug },
      props: { story }
    }))
  );
}

interface Props {
  story: any;
}

const { story } = Astro.props;
const { Content } = await story.render();
---

<BaseLayout locale={locale} title={story.data.title}>
  <article>
    <Content />
  </article>
</BaseLayout>
```

```tsx
// src/components/interactive/MetricsChart.tsx
import { useState } from 'react';
import { LineChart, Line, Tooltip } from 'recharts';

interface Props {
  data: Array<{ month: string; volunteers: number }>;
}

export function MetricsChart({ data }: Props) {
  const [hoveredPoint, setHoveredPoint] = useState<string | null>(null);

  return (
    <LineChart data={data}>
      <Line
        type="monotone"
        dataKey="volunteers"
        onMouseEnter={(point) => setHoveredPoint(point.month)}
        onMouseLeave={() => setHoveredPoint(null)}
      />
      <Tooltip />
    </LineChart>
  );
}
```

```astro

  client:visible
/>
```

```tsx
// src/components/interactive/NewsletterSignup.tsx
export function NewsletterSignup() {
  const [error, setError] = useState<string | null>(null);
  const [success, setSuccess] = useState(false);

  return (
    <form aria-label="Newsletter subscription">
      <div>
        <label htmlFor="email">
          Email address
          <span className="text-red-600" aria-label="required">*</span>
        </label>
        <input
          id="email"
          type="email"
          required
          aria-describedby={error ? "email-error" : undefined}
        />
        {error && (
          <div id="email-error" role="alert" className="text-red-600">
            {error}
          </div>
        )}
      </div>
      <button type="submit" aria-busy={loading}>
        {loading ? 'Subscribing...' : 'Subscribe'}
      </button>
      {success && (
        <div role="status" aria-live="polite" className="text-green-600">
          Successfully subscribed to newsletter!
        </div>
      )}
    </form>
  );
}
```

## Agent Coordination Rules

1. **No specialist does the lead's orchestration** - Leads coordinate, specialists execute
2. **No implementation overlap** - Clear ownership per agent (see When to Invoke)
3. **Dependencies mapped** - Blocked work escalated early (e.g., pages block content publication)
4. **Test coverage required** - No merges without passing `pnpm build` and quality audits
5. **Documentation mandatory** - Every page, component, and change documented in reports
